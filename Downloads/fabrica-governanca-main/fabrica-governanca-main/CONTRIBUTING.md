# Guia de Contribuição


## Fluxo padrão


1. Criar ou assumir uma Issue no board
2. Mover para `In Progress`
3. Criar branch a partir da `main`
4. Implementar
5. Criar Pull Request
6. Aguardar review
7. Merge após aprovação


## Regras


- Um card por vez por aluno
- Commits pequenos e descritivos
- PR obrigatório para `main`
- Seguir padrões definidos em `docs/`


## Estrutura de branch

```
feature/-descricao-curta fix/-descricao-curta
```

## Revisões

- 1 reviewer obrigatório
- Tech Lead revisa mudanças estruturais
